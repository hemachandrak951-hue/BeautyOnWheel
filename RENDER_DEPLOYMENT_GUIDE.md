# Render Deployment Guide - BeautyOnWheel Auth Service

## Prerequisites
- Render account: https://render.com
- Git repository pushed with updated configuration

## Deployment Steps

### 1. Create PostgreSQL Database on Render
1. Go to https://dashboard.render.com
2. Click **"New +"** → **"PostgreSQL"**
3. Configure:
   - **Name**: `beautyonwheel-db`
   - **Database**: `beautyonwheel_prod`
   - **User**: `postgres`
   - **Region**: Choose your preferred region
   - **PostgreSQL Version**: 14
   - **Plan**: Standard or higher
4. Click **"Create Database"**
5. Wait for database to be ready (status shows "Available")
6. Save the connection details - you'll need them

### 2. Create Backend Service on Render
1. Click **"New +"** → **"Web Service"**
2. Connect your GitHub repository
3. Configure:
   - **Name**: `beautyonwheel-auth-service`
   - **Region**: Same as database
   - **Branch**: `main` (or your deploy branch)
   - **Runtime**: `Java`
   - **Build Command**: `mvn clean package -DskipTests`
   - **Start Command**: `java -jar target/beautyonwheel-auth-service-*.jar --spring.profiles.active=prod`
   - **Plan**: Standard or higher

### 3. Set Environment Variables
In the Render Web Service dashboard, add these environment variables:

#### Required Variables:
```
SPRING_PROFILES_ACTIVE=prod
PORT=8080

DB_HOST=<your-database-host>
DB_PORT=5432
DB_NAME=beautyonwheel_prod
DB_USERNAME=postgres
DB_PASSWORD=<your-database-password>

JWT_SECRET=<generate-a-32-char-random-string>
CORS_ALLOWED_ORIGINS=https://yourdomain.com,https://www.yourdomain.com
```

#### How to Generate JWT_SECRET:
```bash
# Generate a random 32+ character string
openssl rand -base64 32
# Or use an online tool: https://www.uuidgenerator.net/
```

#### Example Values:
```
JWT_SECRET=aB3cD5eF7gH9iJ1kL3mN5oP7qR9sT1uV3
CORS_ALLOWED_ORIGINS=https://beautyonwheel.vercel.app,https://www.beautyonwheel.vercel.app
```

### 4. Connect Database to Service
1. In the Web Service, scroll to **"Environment"**
2. Look for the PostgreSQL instance and select it from the dropdown
3. This will automatically populate DB_HOST, DB_PORT, DB_NAME, DB_USERNAME, DB_PASSWORD

### 5. Deploy
1. Click **"Create Web Service"**
2. Render will automatically deploy from your main branch
3. Monitor the logs in the **"Logs"** tab
4. Once deployed, you'll get a URL like: `https://beautyonwheel-auth-service.onrender.com`

## Troubleshooting

### Database Connection Refused
- Check that DB_HOST, DB_PORT, DB_NAME, DB_USERNAME, DB_PASSWORD are set correctly
- Verify the PostgreSQL instance is in "Available" status
- Check that SPRING_PROFILES_ACTIVE=prod is set

### JWT_SECRET Not Set Error
- Set JWT_SECRET environment variable (min 32 characters)
- Redeploy the service after setting it

### CORS Errors
- Update CORS_ALLOWED_ORIGINS to match your frontend domain
- Include both http and https variants if needed
- Separate multiple origins with commas

### Build Failures
- Check the build logs for Maven errors
- Ensure pom.xml is in the root directory
- Verify Java version compatibility (Java 21)

## Updating Deployments

### To deploy changes:
1. Push to your main branch on GitHub
2. Render will automatically redeploy
3. Monitor progress in the Logs tab

### To update environment variables:
1. Go to Web Service → Environment
2. Edit the variable
3. Click "Save"
4. Service will automatically redeploy

## Frontend Configuration

Update your frontend to use the backend URL:

### environment.prod.ts
```typescript
export const environment = {
  production: true,
  apiUrl: 'https://beautyonwheel-auth-service.onrender.com/api/v1'
};
```

Then deploy frontend to Vercel:
```bash
npm run build:vercel
# Push to GitHub
# Vercel will auto-deploy
```

## Security Best Practices

1. **Never commit secrets** - Use environment variables
2. **Rotate JWT_SECRET regularly** - Update on Render
3. **Enable HTTPS** - Render provides free SSL/TLS
4. **Restrict CORS** - Only allow your frontend domains
5. **Use strong database password** - Generated by Render automatically
6. **Monitor logs** - Watch for suspicious activity
7. **Keep dependencies updated** - Regular Maven updates

## Database Backup

Render provides automated backups for Standard and higher plans. To backup manually:

```bash
# Get connection string from Render dashboard
pg_dump "postgresql://user:password@host:port/database" > backup.sql
```
